Introduction

Turbo Stack is the most complete, flexible, and production-ready Docker LAMP & LEMP environment for developers.

Stop wasting time configuring servers. Get everything you need—Apache, Nginx, MySQL/MariaDB, Redis, Varnish, Memcached, Mailpit, and more—in one powerful, optimized Docker setup.

Dual Modes

Hybrid (Nginx + Apache) or Thunder (Nginx + PHP-FPM).

PHP 7.4 - 8.5

Switch PHP versions instantly for any application.

Installation

The stack works on macOS, Linux, Windows, and other Unix-like systems. Ensure you have Docker and Docker Compose v2 installed.

1. Clone the repository

git clone https://github.com/kevinpareek/turbo-stack.git
cd turbo-stack

2. Configure

Run the TBS wizard to choose your PHP version, Database, and Stack Mode.

./tbs.sh config

3. Launch

./tbs.sh start

Core Commands

Manage your entire stack with simple, intuitive commands using the tbs utility.

Command Description
tbs startStart all services & open dashboard
tbs stopStop all services & cleanup
tbs restartRestart stack & apply changes
tbs statusShow status of running containers
tbs infoShow stack and system information
tbs configRun the configuration wizard

App Management

Create and manage multiple applications within the same stack.

./tbs.sh app add my-new-app
Command Description
tbs app add <name>Create a new app (SSH, SSL, Vhost, DB)
tbs app rm <app>Delete an application and its data
tbs app ssl <app>Manage SSL certificates
tbs app php <app>Configure PHP versions for the app

Project Creators

Scaffold popular frameworks instantly with pre-configured settings.

./tbs.sh create laravel my-laravel-app
./tbs.sh create wordpress my-blog

Security & Hardening

Turbo Stack is tuned for local development by default. For production, follow these hardening steps:

  • Environment: Set APP_ENV=production in your .env file.
  • Credentials: Change all default passwords (MySQL, Redis, etc.) to strong, unique values.
  • Xdebug: Disable Xdebug by setting INSTALL_XDEBUG=false.
  • Network: Expose only ports 80 and 443. Keep internal services (DB, Redis) isolated.

Stack Modes

Turbo Stack supports two primary modes of operation:

Hybrid Mode (Default)

Nginx acts as a reverse proxy, Varnish handles caching, and Apache serves the PHP content. Best for compatibility with .htaccess files.

Thunder Mode

Nginx communicates directly with PHP-FPM. This mode offers the highest performance and is recommended for modern applications.
